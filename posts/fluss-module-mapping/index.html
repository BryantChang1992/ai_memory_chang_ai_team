Fluss 所有模块与 Kafka 2.7.2 的完整对照表，包含 100+ 类的逐一对比、独家能力矩阵和代码复用度统计。

7.1 Maven 模块 → Kafka Sub-Project 映射

Fluss Maven Module Kafka 2.7.2 Sub-Project / Package 核心功能 实现方式
fluss-common clients (部分) + connect:api (部分) 公共数据结构、配置、Row 抽象、Arrow 工具、文件系统 自研 + Kafka 复用
fluss-server core 服务器核心：Log/Replica/KV/Coordinator/Tablet 管理 高度复用 Kafka core
fluss-client clients (Producer/Consumer/Admin) Java 客户端：Writer/Scanner/Lookuper/Admin 参考 Kafka Client 架构自研
fluss-rpc clients (network 部分) RPC 框架：Netty 传输、Protobuf 序列化、Gateway 代理 自研（Netty + Protobuf）
fluss-kafka （无对应，新架构） Kafka 协议兼容层 自研（当前骨架阶段）
fluss-flink* （外部，在 Flink 项目中） Flink Source/Sink/Catalog/Tiering 自研（符合 Flink Connector API）
fluss-lake-* （无对应） Lake Storage 插件：Iceberg/Paimon/Hudi/Lance 自研
fluss-protogen （无对应） Protobuf 代码生成 自研
fluss-filesystems （外部插件） 文件系统插件（S3/HDFS/OSS） 自研
fluss-metrics clients (metrics) 指标收集与上报 参考 Kafka 自研
fluss-jmh jmh-benchmarks JMH 性能基准测试 同架构
fluss-dist tools (分布部分) 发行版打包 同架构

能力 Fluss Kafka 2.7.2 Fluss 优势

ISR 复制 ✅ ✅ 持平

Consumer 组 ❌ (用 Scanner 自管理) ✅ Kafka 更标准

幂等写入 ✅ (简易) ✅ (Producer ID + Txn) Kafka 更完善
Connect 框架 ❌ ✅ Kafka 生态更强
Streams DSL ❌ (用 Flink) ✅ 各有生态
Primary Key 表 ✅ (RocksDB) ❌ Fluss 独有
Upsert / Delete ✅ ❌ Fluss 独有
KV 点查 ✅ ❌ Fluss 独有
Merge Engine ✅ (Aggregate/PartialUpdate/Versioned) ❌ Fluss 独有
Arrow 列式 ✅ ❌ Fluss 独有
列裁剪 ✅ ❌ Fluss 独有
谓词下推 ✅ ❌ Fluss 独有
Schema 管理 ✅ (内建) ❌ Fluss 独有
Table 概念 ✅ (Database → Table) ❌ (仅 Topic) Fluss 独有
自动分区 ✅ ❌ Fluss 独有
远程分层存储 ✅ (原生) ❌ (KIP-405 后) Fluss 原生

存算分离 ✅ (TabletServer 独立) ❌ (Broker 存算耦合) Fluss 更灵活
ZK 依赖 ✅ ✅ 持平

7.4 代码复用程度统计

复用类型 文件数（估计） 示例
直接复用（标注 “based on Kafka”） ~15-20 LocalLog, LogTablet, LogManager, ReplicaFetcherThread, DelayedOperation, OffsetCheckpointFile
参考实现（架构相似但重写） ~30-40 ReplicaManager, Sender, RecordAccumulator, BucketAssigner
完全自研 ~200+ 整个 KV 子系统、RPC 框架、客户端 API、Lake Storage、Flink 集成

Fluss 总体：~70% 自研 + ~30% Kafka 复用/参考

核心复用集中在 Log Segment 管理 和 Replica 复制基础框架，差异化最大的 KV 存储、Lake 集成、Arrow 列式、客户端 API 全部为自研。

               Kafka 2.7.2                Fluss
               ──────────                ─────
数据模型:      Topic (key+value bytes)    Table (typed schema)
存储引擎:      本地 Log Segment           本地 Log + KV (RocksDB) + Remote Lake
记录格式:      Kafka Record (行式)        Arrow (列式, 3种 LogFormat)
API:          Producer/Consumer          Writer/Scanner/Lookuper
协调:         Controller (内嵌)           CoordinatorServer (独立)
计算:         Connect + Streams          Flink/Spark Source+Sink + Catalog
协议:         单一 Kafka 协议             Fluss 协议 + Kafka 协议(骨架)
湖仓:         无                         Iceberg/Paimon/Hudi/Lance
分层存储:     无 (KIP-405后引入)          原生 RemoteLog + Lake Tiering
