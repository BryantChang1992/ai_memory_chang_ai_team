## 领域记忆结构

每个领域应包含：

```
memory/domains/<领域名>/
├── architecture.md      # 架构设计
├── best-practices.md    # 最佳实践
├── runbooks.md          # 运维手册/操作指南
├── faq.md               # 常见问题
└── resources.md         # 工具/资源清单
```

## 领域专家职责

1. **维护领域知识**
   - 持续更新最佳实践
   - 记录技术方案选型
   - 编写运维手册

2. **支持项目**
   - 为项目提供专业建议
   - Review 项目技术方案
   - 参与项目复盘

3. **培养团队**
   - 培训执行层 worker
   - 分享领域知识
   - 建设 FAQ 库

---

## 新增领域流程

1. VP 提出领域需求
2. CEO 批准
3. 任命领域专家
4. 创建领域目录
5. 初始化领域记忆

---

**领域体系是演化的，根据业务发展持续调整。**
