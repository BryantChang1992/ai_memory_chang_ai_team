## 2026-04-19: 多飞书机器人架构决策

**决策者：** CEO + Bryant  
**影响范围：** 公司级  
**状态：** ✅ 已实施

### 决策内容
采用单实例多飞书账号方案，实现多 agent 架构。

### 背景
需要构建 VP 层、专家层、执行层三层 agent 架构。

### 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| 单实例多账号 | 成本低、易管理 | 需要配置 bindings |
| 多实例 | 完全隔离 | 资源消耗大、通信复杂 |

### 最终决策
✅ 选择单实例多账号方案
