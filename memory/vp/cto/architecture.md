## 当前架构

### 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    OpenClaw 单实例                        │
├─────────────────────────────────────────────────────────┤
│  VP 层 (常驻)     │  专家层 (常驻)    │  执行层 (动态)     │
│  - CEO (main)     │  - Infra 专家     │  - RD worker      │
│  - CTO (cto-agent)│  - Perf 专家      │  - SRE worker     │
│  - CPO (待招募)    │  - SRE 专家       │  - QA worker      │
└─────────────────────────────────────────────────────────┘
```

### 飞书账号路由

```
飞书消息
    │
    ├── CEO 账号 (cli_a96e7225c1f89bb3) → main agent
    └── CTO 账号 (cli_a96e6494a3b89bc4) → cto-agent
```

### 记忆体系

```
memory/
├── shared/           # L1 公司级
├── vp/               # L2 VP 级
├── projects/         # L2 项目级
├── domains/          # L2 领域级
└── sessions/         # L3 会话级（压缩同步）
```

## 待优化

### 1. 配置热更新

**问题：** 配置变更需要重启 Gateway。

**方案：**
- 研究 OpenClaw 配置热加载
- 或实现配置变更自动重启脚本

**优先级：** 中

---

### 2. Token 监控自动化

**问题：** 手动监控 Token 使用，容易超预算。

**方案：**
- 创建 monitor-tokens.sh 脚本
- 设置阈值告警
- 自动裁剪超限记忆

**优先级：** 高

---

### 3. 会话压缩算法

**问题：** 当前压缩策略简单，可能丢失关键信息。

**方案：**
- 研究 AI 辅助摘要生成
- 保留决策点/待办/关键上下文
- 建立压缩质量评估

**优先级：** 中
