Apache Iceberg 1.11.0 正式发布 — V3 规范生产就绪
🚀 重大更新
📅 2026-05-19 · Apache Iceberg Releases · 深度解读

Apache Iceberg 1.11.0 于 5 月 19 日发布，是一次结构性里程碑更新，核心变化包括：

  • Manifest List 加密（GCM）：采用 Galois/Counter Mode 流密码对 Manifest List 进行加密，支持并行随机访问解密，查询引擎可按需解密特定区块，解决了"数据加密但元数据明文"的安全缺陷。这在金融合规和医疗 HIPAA 场景下意义重大。
  • 可插拔 File Format API：将 Parquet/ORC/Avro 的硬编码路径抽象为插件式接口，允许社区接入 Vortex（Parquet 继任者，支持 SIMD 过滤）和 Lance（面向 ML 的列式格式）等下一代格式，为 AI 训练场景的格式创新铺路。
  • V3 规范稳定化：半结构化数据类型、空间坐标（geometry/geography）进入默认特性集，SQL UDF 规范正式引入。
  • 平台清理：停止支持 Java 11、Spark 3.4 标记弃用、Flink 1.19 移除，全面转向 Java 17 和 Spark 4.x。
  • 引擎支持：同时支持 Spark 4.1、4.0、3.5 和 Flink 2.1、2.0、1.20 多版本运行时。
Iceberg Summit 2026 — V4 路线图公布
🔮 路线图
📅 2026-05 · Iceberg Summit 2026

在 Iceberg Summit 2026 上，核心维护者公布了 Apache Iceberg v4 路线图方向：

  • 新一代 REST Catalog 能力扩展，增强多引擎互操作和细粒度权限控制
  • 改进的 Compaction 机制，面向高频写入场景优化
  • 扩展引擎互操作性，降低跨计算引擎的元数据延迟
Delta Lake 4.2.0 — Catalog-Managed Table 体系完善
📦 版本发布
📅 2026-04-16 · GitHub Release · 开发者解读

Delta Lake 4.2.0 于 4 月 16 日发布，构建于 Apache Spark 4.1.0 / 4.0.1 之上，亮点包括：

  • Catalog-Managed Table 增强：支持 REPLACE TABLE / RTAS（原子操作）、Dynamic Partition Overwrite，表创建时自动同步 Schema/属性至 Catalog
  • Delta Spark V2 Streaming Read（实验性）：支持 startingTimestamp、skipChangeCommits 等关键选项，补齐流式读取能力
  • Kernel-based Flink 连接器（实验性）：基于 Delta Kernel 的全新 Flink 连接器，使 Flink 可读写 Catalog-Managed Delta 表
  • Geospatial & Variant GA：geometry/geography 类型支持 Bounding-Box Data Skipping；Variant 列正式 GA，成为半结构化数据默认格式
  • Server-Side Planning OAuth：扫描计划委托至 Catalog 服务器时支持 OAuth 认证
  • INSERT BY NAME + Schema Evolution：SQL 插入语句支持自动 Schema 演化
  • 安全加固：多表面安全审计、依赖扫描，主动降低供应链风险
Delta Lake 4.1.0 — Catalog-Managed Table 功能落地
📦 版本发布
📅 2026-03-01 · 官方博客

3 月发布的 Delta Lake 4.1.0 为 4.2 版本奠定了基础：

  • Catalog-Managed Table 生产支持：建立以 Catalog 为协调者的表访问模式，提交协调从文件系统迁移至 Catalog
  • Server-Side Planning（预览）：将扫描计划（文件发现、谓词过滤、凭证配置）委托至 Catalog 服务器，核心动机是细粒度访问控制（FGAC）
  • 原子 CTAS：CREATE TABLE AS SELECT 对 Unity Catalog 表实现完全原子化
  • 冲突自由特性启用：可在不阻塞并发写入的情况下启用 Deletion Vectors 和 Column Mapping
  • 正式放弃 Spark 3.5 支持，全面转向 Java 17 和 Spark 4.x
Snowflake Iceberg v3 表格式全面 GA
📦 GA 发布
📅 2026-05-07 · Snowflake Release Notes

Snowflake 于 5 月 7 日正式宣布 Apache Iceberg v3 表规范的全面 GA，成为三大云数仓中首个完成 V3 适配的厂商：

  • 支持 V3 数据类型：geography、geometry、nanosecond timestamp、variant
  • 默认值（Default Values）：支持为 Iceberg 表列定义默认值
  • 删除向量（Deletion Vectors）：提升更新/删除操作的性能
  • 行级血统（Row Lineage）：追踪 CDC 变更的行级血统信息
  • Horizon REST Catalog API：外部引擎可通过该 API 读取 Snowflake 管理的 Iceberg v3 表（外部写入暂不支持）
  • GET_DDL 函数对所有 Iceberg 表返回 ICEBERG_VERSION 属性
Apache Hudi 1.2.0-SNAPSHOT — RFC-100 非结构化数据存储提案
🔮 提案
📅 2026-05-23 · GitHub Releases · RFC-100 PR

Apache Hudi 社区活跃推进 1.2.0 开发，本周最值得关注的是 RFC-100: Unstructured Data Storage in Hudi（初始 Strawman 提案），由 @vinothchandar 提交。这是 Hudi 首次系统性提出在湖仓框架内管理非结构化数据（图片、视频、音频、文档）的方案，直接面向多模态 AI 场景。

其他活跃改进：

  • 二级索引修复：正确处理 secondary index 中的更新和删除操作
  • Flink 改进：MDT compaction 优化、stream read key selector 修复
  • Spark Schema Evolution：修复嵌套列的 Schema 演化问题
  • 性能优化：降低 HFile log block 的内存占用
  • 新 Demo 应用：hudi-notebooks 用于交互式体验

Delta Kernel 跨引擎连接器矩阵形成
📈 趋势
📅 2026-Q2

Delta Lake 4.1/4.2 版本通过 Delta Kernel 实现了跨引擎集成能力的质变：

  • Spark：V2 DataSource 连接器，支持 Catalog-Managed Table 的流式读写
  • Flink：全新 Kernel-based Flink 连接器（实验性），读写 Catalog-Managed 表
  • Delta Connect：Server-Side Planning + OAuth，将查询计划委托至 Catalog 服务器，降低客户端开销
  • 这一架构使 Delta 从"Spark 专属格式"演变为真正的多引擎湖仓格式，与 Iceberg 的 REST Catalog 生态形成对标

Hudi RFC-100：湖仓原生非结构化数据存储
🚀 突破性提案
📅 2026-05-23 · RFC-100 (GitHub #13924)

这是本周最值得关注的提案之一。Apache Hudi RFC-100 首次在湖仓框架内系统性设计了非结构化数据（图片、视频、音频、文档）的存储方案。其核心意义在于：

  • 将湖仓从"结构化/SQL 数据"的边界扩展到多模态领域
  • 为 AI 训练场景提供统一的、带版本管理和增量处理能力的非结构化数据存储
  • 结合 Hudi 已有的增量处理框架，可构建端到端的"数据摄入 → 非结构化存储 → AI 训练"流水线

该提案目前处于 Strawman（草案）阶段，但方向性意义重大，值得持续跟踪。

Iceberg 1.11 + Delta 4.2：Geospatial 类型 GA
📈 趋势
📅 2026-05

Iceberg 1.11 和 Delta 4.2 在不同路径上同时推进了对复杂数据类型的支持：

  • Iceberg：在 V3 规范中明确了 geometry 类型的限制，增加了 geospatial bounding box API 用于查询优化
  • Delta Kernel：geometry/geography 类型正式 GA，支持 Bounding-Box Data Skipping 进行空间查询加速
  • Variant 类型：Delta 4.2 中 Variant 列正式 GA，Variant Shredding 为高频字段提供准列式读取性能，成为半结构化事件数据的推荐存储格式
  • 这标志着开放表格式在 AI 特征存储、时空数据分析等场景的能力边界正在快速扩展
下一代文件格式：Vortex 与 Lance 进入 Iceberg 生态
🔮 预览
📅 2026-05 · Iceberg 1.11 深度解读

Iceberg 1.11 的 File Format API 为下一代文件格式打开了生态入口：

  • Vortex：由 Apache Arrow/DataFusion 社区推动的 Parquet 继任者，采用定宽列 + Bitmap Null Mask，支持 SIMD 指令直接在内存映射文件上进行过滤，无需 CPU 解压周期。社区正基于新 API 构建 Vortex-backed Iceberg 插件
  • Lance：专为 ML/AI 工作负载设计的列式格式，在随机访问和向量化读取方面有先天优势，适合嵌入向量和特征存储场景

对 AI 数据平台的意义：这些格式专为高性能分析和 ML 训练设计，将显著提升 AI 训练管道的 I/O 效率。
