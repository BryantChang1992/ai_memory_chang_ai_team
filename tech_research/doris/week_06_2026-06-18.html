一、Apache Doris / OLAP 分析型数据库

1. Apache Doris 4.0.6 正式发布

Doris 4.0.6 正式发布，紧随 5月24日的 4.1.1 和 4月21日的 4.1.0 之后。 GitHub releases 页面显示后续还有持续 commit 合并（最近日 commit 在 6月17日）。

#release#ApacheDoris#OLAP

2. VeloDB + Apache Doris 通过 Supermetal 实现单二进制 CDC

Supermetal 现在可将 Postgres、MySQL、MongoDB、SQL Server、Oracle 五种源的操作数据实时复制到 Apache Doris/VeloDB Cloud。 架构从 Debezium→Kafka→Flink→Doris 四跳简化为 Supermetal→Object Store→Doris 两跳，全程 Parquet + Arrow 列式传输，省去 JVM 依赖。 基准测试：433M 行快照仅需 6 分 11 秒（~1.5M rows/sec），CDC 端到端延迟在 25K ops/sec 负载下保持在 7-9 秒。 Merge-on-Write 模型保证查询零读时去重。

#CDC#VeloDB#Supermetal#ApacheDoris#实时同步

3. Apache Doris 2026 Roadmap 更新

2026年路线图以 "Scale Intelligence, Accelerate Insights" 为主题，聚焦三大方向： (a) AI & Hybrid Search — 磁盘ANN支持单表100亿向量、全文搜索打分/多索引、Iceberg混合搜索； (b) 存储引擎 — 超大Tablet(100GB+)、存算分离弹性调度、分布式缓存； (c) 数据湖集成 — Iceberg V3全面支持、Paimon读写、Arrow Flight Data Catalog。 MERGE INTO、递归CTE、ASOF JOIN、Python UDF 等新特性也在规划中。

#Roadmap#向量搜索#存算分离#Iceberg#Paimon

二、时序数据库

4. InfluxDB 3.10 发布：Performance Beta 扩展 + 企业级特性

InfluxDB 3.10 核心亮点：(a) Pacha-Tree 性能 Beta 继续扩展，新增端到端备份恢复、行级删除、Parquet 批量导入； (b) 跨数据库插件查询 — Processing Engine 插件可跨库读取写入，实现 raw→rollup→forecast 管道； (c) 生产就绪端点 /ready — 验证对象存储可达性； (d) 并行 Compaction 加速； (e) 用户认证+RBAC 预览（JWT/OAuth/OIDC，Admin/Auditor/Member 三种内置角色）； (f) Catalog v3 自动迁移（on-disk ~5-6x 压缩）。InfluxDB 3 Core 保持 MIT/Apache 2 开源。

#InfluxDB#时序数据库#RBAC#PachaTree#跨库查询

5. InfluxDB 3 合成时序数据生成插件

InfluxDB 3 新增两个 Processing Engine 调度插件：Bird Tracking Simulator（鸟类移动遥测模拟）和 Signal Generator（可配置波形生成器，支持 sine/square/triangle/sawtooth/noise/spike）。 可直接通过 influxdb3 create trigger 一键启动数据生成，用于仪表盘测试、告警验证、边缘复制评估等场景。

#InfluxDB#时序数据#插件#测试工具

6. TimescaleDB 2.28.0 发布：压缩查询加速 + 增量刷新

TimescaleDB v2.28.0 关键更新：(a) first()/last() 直接利用压缩列存批量元数据，跳过批量解压，最新值查询显著加速； (b) refresh_continuous_aggregate() 支持增量分批刷新，锁粒度大幅降低； (c) 向量化执行引擎覆盖 CASE 表达式，条件聚合不再退化为逐行解压； (d) ALTER MATERIALIZED VIEW ADD COLUMN 支持无重建添加新聚合列； (e) 移除自适应 Chunking。同时宣布 2.28.x 为最后一个支持 PG15 的版本。

#TimescaleDB#时序数据库#压缩优化#连续聚合#PostgreSQL

7. InfluxDB 3 卫星遥测 + ITAR 数据驻留架构

探讨卫星任务运营中如何用 InfluxDB 3 构建受控遥测架构。 Ingress 支持 Telegraf Agent、MQTT Pipeline 等路径，Tag 注入操作上下文（航天器ID、子系统、地面站）； 部署层面 InfluxDB 3 Core 自托管满足边缘/私有云需求，Enterprise 提供 HA + 读副本分离。 核心论点：共享时序架构消除数据孤岛，让工程师在异常响应时无需跨系统拼凑数据。

#InfluxDB#遥测#ITAR#架构

三、存储引擎 / 分布式共识

8. CockroachDB @ SIGMOD 2026：可扩展 Leader Lease 论文

Cockroach Labs 在 SIGMOD 2026 发表论文 "Scalable Leader Leases For Multi Consensus Groups in CockroachDB"。 解决多 Raft Group 场景下 Leader Lease 机制的可扩展性问题——分布式 SQL 数据库实现低延迟一致性读的关键基础设施。

#分布式共识#Raft#Lease#SIGMOD2026#CockroachDB

四、云原生数据库架构

9. CockroachDB：Agentic AI 架构中的数据库角色

探讨当企业将自主 AI Agent 连接到生产数据栈时，数据库需要提供 Memory（持久化记忆）、Context（上下文管理）和 Control（访问控制）三个能力。 CockroachDB 作为分布式 SQL 数据库在 Agentic AI 架构中的定位。

#AgenticAI#云原生#分布式数据库#CockroachDB

10. SingleStore：云数据库客户自主管理加密密钥 (CMEK)

SingleStore 发布云数据库加密控制频谱，重点介绍 Customer-Managed Encryption Keys (CMEK) 能力。 作为云原生数据库安全架构的关键演进，CMEK 让企业客户在共享责任模型中获得对加密密钥的完全生命周期控制。
