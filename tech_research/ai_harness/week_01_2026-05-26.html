🤖 OpenAI: Codex 生态全面铺开

🏆 Codex 被 Gartner 评为企业级编程 Agent 领导者

2026 年 5 月 22 日，OpenAI 宣布 Codex 被 Gartner 评为企业级编程 Agent 领域的 Leader。 Codex 目前每周被超过 400 万开发者 使用，合作伙伴生态涵盖 Cisco、Datadog、Dell 和 NVIDIA 等企业级平台。 这一里程碑标志着 AI 编程 Agent 从开发者工具正式进入企业关键基础设施层。

与单纯代码补全工具不同，Codex 定位为完整的 Agent Harness——它管理云端模型与本地开发环境之间的交互， 支持 CLI、IDE 插件和桌面应用三种形态，可以自主执行从读文件到创建 Git 分支的全栈开发操作。

来源: OpenAI Blog — Gartner Leader · 2026-05-22

📱 Codex 移动端 + 远程 SSH + Hooks 全线发布

OpenAI 正式推出 ChatGPT 移动应用中的 Codex 体验，开发者可以通过手机随时随地连接正在运行的 Codex 实例， 查看进度、审批命令、切换模型或发起新线程。底层依赖一套安全的 Relay 中继层，在不暴露设备到公网的前提下保持实时状态同步。

关键更新：

NEW Remote SSH GA — Codex 桌面应用可自动发现 SSH 配置中的远程主机，在远程开发环境中创建项目并运行线程，与本地体验一致。
NEW Hooks GA — 通用可用的钩子系统，支持扫描 Prompt 中的密钥、运行校验器、记录对话日志、创建记忆或按仓库/目录自定义 Codex 行为。
NEW Programmatic Access Tokens — 企业版和商业版可创建作用域凭证，用于 CI 流水线和内部自动化。
SAFETY HIPAA 合规 — ChatGPT Enterprise 工作区可在本地环境中合规使用 Codex，覆盖医疗健康场景。

来源: OpenAI — Work with Codex from anywhere · 2026-05

🛡️ Codex Windows 沙箱技术揭秘

OpenAI 工程师团队发表了一篇深度技术文章，详细介绍了为 Codex 在 Windows 平台上构建安全沙箱的过程。 Windows 用户此前只能在"几乎每条命令都需批准"和"完全放开权限"之间二选一，团队的目标是在不要求管理员权限的前提下， 限制 Codex 的文件写入和网络访问能力。

团队评估了三种现有方案并全部否决：AppContainer（隔离太强，不适配开放式的开发者工作流）、 Windows Sandbox（轻量 VM，但无法操作用户实际的工作目录，且不支持 Windows Home SKU）、 Mandatory Integrity Control（会修改宿主机文件系统的信任模型，副作用太大）。 最终方案利用了 SID（安全标识符）+ Write-Restricted Token 的组合， 创建一个名为 sandbox-write 的合成 SID，通过 ACL 精确控制写入范围， 同时保护 .git、.codex 和 .agents 目录不被修改。

来源: OpenAI — Building Codex Windows Sandbox · 2026-05

🧬 Anthropic: Agent 工程方法论密集输出

📦 Agent Skills 成为开放标准

Anthropic 将 Agent Skills 作为一个开放标准发布（agentskills.io）， 定义了一套轻量级的格式，使 Agent 能够动态发现和加载领域特定的专业知识。 Skills 的核心设计原则是渐进式披露（Progressive Disclosure）：

第一层 — Agent 启动时仅预加载所有 Skills 的名称和描述到系统提示词中
第二层 — Agent 判断某个 Skill 相关时，读取完整的 SKILL.md
第三层 — 可按需从 Skill 目录中加载辅助文件和可执行脚本

Skills 已全面支持 Claude.ai、Claude Code、Claude Agent SDK 和 Claude Developer Platform。 Anthropic 还公布了 Skills 安全指南，强调仅从可信来源安装并进行审计检查。 展望未来，团队正在探索让 Agent 自行创建、编辑和评估 Skills，实现行为模式的可复用编码化。

来源: Anthropic Engineering — Agent Skills · 2025-12-18

🏗️ 长运行 Agent 的 Harness 设计范式

针对"如何让 Agent 跨多个上下文窗口持续有效工作"这一开放问题，Anthropic 发布了一套基于 Claude Agent SDK 的双阶段 Harness 设计模式：

阶段一：Initializer Agent — 首个会话使用专门 Prompt，创建 init.sh 环境脚本、 claude-progress.txt 进度文件，并生成包含 200+ 特性 的 JSON 需求列表（所有特性初始标记为 "passes": false）。

阶段二：Coding Agent — 后续每个会话（压缩/重新启动后）聚焦于一个特性的增量实现， 完成后更新 JSON 的 passes 字段、撰写 Git commit 并更新进度文件。

该方法成功解决了两个关键失败模式：(1) Agent 试图"一把梭"完成所有功能导致上下文溢出； (2) 后续 Agent 实例看到已有进展后过早宣布任务完成。 核心洞察是：让每个 Agent 会话结束时环境处于"可合并到主分支"的整洁状态。

来源: Anthropic Engineering — Effective Harnesses for Long-Running Agents · 2026-03-24

📊 Agent 评估体系指南

Anthropic 发布了迄今最全面的 Agent 评估方法论文章，系统化定义了 Agent 评估的核心组件：

概念定义
Task（任务）带有明确输入和成功标准的单次测试
Trial（试验）对同一任务的多次运行，控制输出随机性
Grader（评分器）包含多个断言的评分逻辑，分三类：代码级、模型级、人工级
Transcript（轨迹）完整记录：输出、工具调用、推理过程、中间结果
Outcome（结果）环境中的最终状态（区别于 Agent 的口头声明）
Eval Harness端到端运行评估的基础设施
Agent Harness使模型具备 Agent 能力的 Scaffold 系统

文章特别强调了一个微妙陷阱：Agent 说"你的航班已预订"不等于数据库中真的存在预订记录。 评估必须区分 Transcript（Agent 说了什么）和 Outcome（实际发生了什么）。 文章还分享了 Claude Code、Descript 和 Bolt 等团队在评估实践中的真实案例。

来源: Anthropic Engineering — Demystifying Evals for AI Agents · 2026-05

🔒 Claude Code Auto Mode: 权限跳过的安全之道

Anthropic 介绍了 Claude Code 的 Auto Mode——一种更安全地处理权限跳过的方式。 这是对传统"全部批准"与"全部拒绝"二元模型的突破， 允许 Agent 在受约束边界内自主决策某些操作，同时保持关键操作的人类审批环节。

来源: Anthropic Engineering — Claude Code Auto Mode · 2026-03-25

🔬 Managed Agents: 脑-手分离架构

Anthropic 深度探讨了大规模 Managed Agent 的架构模式——"解耦大脑与双手"： 将 Agent 的推理层（模型/规划）与执行层（工具调用/环境交互）分离， 使 Agent 能够在不暴露底层模型细节的情况下安全地大规模运行。 这一架构对于企业级 Agent 部署具有重要参考价值。

来源: Anthropic Engineering — Scaling Managed Agents · 2026-04-08

🔌 Agent 框架与协议生态

LangChain 于 5 月 14 日正式发布 LangChain Labs，定位为 Agent 创新的孵化器， 同时推出 deepagents——一个专门面向长时间运行、复杂任务的 Agent 框架。 deepagents 的设计理念与 Anthropic 的 Harness 方法论形成互补：提供开箱即用的长运行支持， 包括上下文窗口管理、状态持久化和任务分解。

🌐 MCP: 协议生态持续扩展

Model Context Protocol (MCP) 目前已获得行业广泛支持，包括 Claude、ChatGPT、 VS Code、Cursor 和 MCPJam 等主流 AI 客户端。最新规范版本（2025-11-25）定义了三大服务器能力 （Resources / Prompts / Tools）和三项客户端能力（Sampling / Roots / Elicitation）。

Elicitation（引导式交互） 是值得关注的新特性——允许服务器在工具执行过程中向用户请求补充信息， 这为 Agent 与人类的交互模式提供了标准化框架，而非依赖各实现方自行定义。 MCP 整体的安全模型强调用户同意、数据隐私和工具安全三大原则。

来源: MCP Specification 2025-11-25

🛡️ Agent 安全与评估前沿

五大趋势观察

1. 沙箱化执行成为标准配置。 OpenAI Codex 的 Windows Sandbox 工程实践和 Anthropic 的权限模型演进 表明，Agent 安全已从"信任模型"转向"OS 级强制隔离"。Codex 团队甚至开发了自定义 Windows 沙箱， 因为现有系统方案均无法满足"无需管理员权限+操作真实工作目录+精确文件写入控制"的三重约束。

2. 评估体系从单一维度走向多层综合。 Anthropic 提出的代码级/模型级/人工级三类评分器体系 为 Agent 评估提供了系统化框架。关键教训包括：必须区分 Agent 的"口头输出"与"实际结果"、 多轮试验以控制随机性、以及评估必须覆盖完整轨迹而非仅看最终状态。

3. Skills/能力模块化成为行业共识。 Anthropic 的 Agent Skills 开放标准、 CrewAI 的 Skills Repository、以及 LangChain 的 deepagents 框架都在朝同一方向演进： 将 Agent 能力打包为可组合、可复用、可安全审计的模块。

4. 长运行 Agent 需要专门的 Harness 架构。 "Initializer + Incremental Worker"模式 以及环境状态管理（进度文件/Git/特性列表）正在成为跨上下文窗口运行的行业范式。 单纯的上下文压缩（compaction）不足以解决长运行问题。

5. 企业级部署需求驱动协议标准化。 MCP 的广泛采纳、Codex 的企业合规认证（HIPAA）、 以及各框架的安全审计（如 CrewAI 的 GHSA 修复）表明 Agent 基础设施正在经历从 "开发者实验工具"到"企业关键基础设施"的过渡。
