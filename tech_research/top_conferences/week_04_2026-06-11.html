📈 趋势洞察与总结

趋势一：LLM 推理成为数据库系统新的核心负载
SIGMOD 2026 出现了多篇专门研究 LLM 推理服务的系统论文（AlignedServe、CoDec、Efficient LLM Serving for Agentic Workflows），数据库社区正式将 LLM 推理视为一等公民工作负载。前缀共享、批处理调度、KV Cache 管理等数据库技术正在向推理服务迁移。
趋势二：存储分离架构走向成熟
"Reducing Tail Latency in Storage-Disaggregated Database Systems" 和 "CMANNS" 等论文表明，计算-存储分离已从概念验证进入生产调优阶段。尾延迟优化、GPU 加速索引构建等后分离时代的工程问题成为研究热点。
趋势三：向量/图数据基础设施深度融合
ANNS（近似最近邻搜索）和图查询正从独立组件向数据库内核融合——PostgreSQL 上的向量搜索研究、GPU 加速的正则路径查询、以及拓扑感知的图存储系统，标志着"多模态数据库"时代加速到来。
趋势四：LLM for DB 从工具走向平台
DBugScribe（Bug 复现）、Test Oracle Discovery（测试预言）、Database-Native Function Synthesis（代码合成）——LLM 不再只是数据库的辅助工具，而是在重塑数据库的测试、开发和运维全流程。
趋势五：持久化索引结构迎接新硬件
FM-Index 变体、持久化 ART、并发路径复制——面向 NVM/CXL/PMem 的存储引擎设计正从"适配"走向"原生"。CMANNS 的 Compute-Memory Disaggregation 方案是 GPU 与存储融合的典型代表。
