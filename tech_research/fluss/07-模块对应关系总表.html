7.4 代码复用程度统计


复用类型文件数（估计）示例
直接复用（标注 "based on Kafka"）~15-20LocalLog, LogTablet, LogManager, ReplicaFetcherThread, DelayedOperation, OffsetCheckpointFile
参考实现（架构相似但重写）~30-40ReplicaManager, Sender, RecordAccumulator, BucketAssigner
完全自研~200+整个 KV 子系统、RPC 框架、客户端 API、Lake Storage、Flink 集成

Fluss 总体：~70% 自研 + ~30% Kafka 复用/参考


核心复用集中在 Log Segment 管理 和 Replica 复制基础框架，差异化最大的 KV 存储、Lake 集成、Arrow 列式、客户端 API 全部为自研。



7.5 架构对比总结



               Kafka 2.7.2                Fluss
               ──────────                ─────
数据模型:      Topic (key+value bytes)    Table (typed schema)
存储引擎:      本地 Log Segment           本地 Log + KV (RocksDB) + Remote Lake
记录格式:      Kafka Record (行式)        Arrow (列式, 3种 LogFormat)
API:          Producer/Consumer          Writer/Scanner/Lookuper
协调:         Controller (内嵌)           CoordinatorServer (独立)
计算:         Connect + Streams          Flink/Spark Source+Sink + Catalog
协议:         单一 Kafka 协议             Fluss 协议 + Kafka 协议(骨架)
湖仓:         无                         Iceberg/Paimon/Hudi/Lance
分层存储:     无 (KIP-405后引入)          原生 RemoteLog + Lake Tiering
