与 Apache Kafka 2.7.2 架构对照

CHANG_AI_TEAM CTO · 2026-06-10 · Fluss trunk vs Kafka 2.7.2 (37a1cc3) · 7 个模块

🔑 核心发现

  1. Fluss 约 30% 代码复用 Kafka（Log Segment 管理 + Replica 复制框架），70% 自研（KV 存储、Arrow 列式、Lake 集成、RPC 框架）
  2. 最大差异化：KV 存储（RocksDB）、Arrow 列式记录格式、原生 Lakehouse 集成（Iceberg/Paimon/Hudi/Lance）
  3. Kafka 兼容层处于骨架阶段：KafkaRequestHandler 中绝大部分 API 处理器为空方法 {}，仅 API_VERSIONS 完整
  4. 存算分离架构：CoordinatorServer 独立进程 vs Kafka Controller 内嵌 Broker，可独立高可用
  5. Lake Tiering：原生 Flink 作业将实时数据持续写入 Lakehouse，实现 Streaming ↔ Lakehouse 一体化
