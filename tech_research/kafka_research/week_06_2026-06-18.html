🔥 本周亮点

AutoMQ 1.7.0 正式发布 🚀

AutoMQ 发布 1.7.0 版本，对比 1.6.5 包含 40+ PR。关键特性：Namespace 支持（多实例隔离）、基于 Protobuf 的 Cluster Events Framework、Autoscaling 指标暴露、Fetch Listener Callbacks 异步 inter-broker 请求、QuorumControllerExtension 生命周期钩子、首次 Maven Snapshot 发布。修复 S3 并发写入异常、LogCache 双重释放竞争、DataBlockIndex int overflow 导致 compaction 数据丢失等关键问题。E2E 测试全面加固 MirrorMaker、Connect、Streams。

Release1.7.0NamespaceAutoscalingCluster Events

Gravitee Kafka 2026 全景分析：KRaft 迁移与 4.x 治理层挑战

Gravitee 6/15 发布 Kafka 2026 全景分析，涵盖 4.x 系列 KRaft 迁移、KIP-932 Share Groups 替代传统消息队列、Tiered Storage 成熟度评估。重点提醒：ZooKeeper 在 4.x 中已完全消除（非 deprecated），3.x 集群必须完成 KRaft 迁移后才能升级。面向 CTO/平台负责人的决策参考。

KRaft MigrationKIP-932Tiered StoragePlatform Decision

Apache Fluss 持续高频迭代：Lake Storage 正确性与 KV 稳定性

Fluss 本周 5 个活跃 PR，焦点集中在 Lake Storage 正确性（snapshot 校验和 watermark 上报到 Paimon）、KV 存储稳定性（RocksDB L0 背压机制）和安全加固（敏感配置脱敏）。FIP-37 bitmap 标量函数增强 bitemporal 分析能力。仍在 incubating 阶段但 PR 质量和数量均在线。

Apache Fluss (Incubating)

FLUSS-3483: Validate lake snapshot offsets before commit

Jackeyzhe 6/18 提交：修复表 drop 后重建导致的 stale V2 lake snapshot commit 竞争条件。新表可能指向旧 tableId 的 offsets metadata，导致 lake tiering 反复失败。PR 在 commit 前校验 tableId、offsets 可读性和 TableBucket 一致性。
