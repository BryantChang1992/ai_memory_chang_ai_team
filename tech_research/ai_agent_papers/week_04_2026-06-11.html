🤝 多 Agent 协作与编排 4 篇

Decentralized Multi-Agent Systems with Shared Context
Jun 2026 2026-06-09
提出一种去中心化的多 Agent 架构，通过 Shared Context 机制实现 Agent 间的隐式信息交换，无需中心化编排器即可协作完成复杂推理任务。 相比传统中心化 MAS，在并行度和容错性方面有显著提升，为大规模 Agent 系统提供了新的架构选择。
🔗 arxiv 去中心化 MAS Shared Context
DarkAgents: A Decentralized Multi-Agent Framework for Trustworthy and Collaborative Task Solving
Jun 2026 2026-06
基于区块链的去中心化多 Agent 框架，通过链上信用机制和共识协议实现无信任环境下的 Agent 协作。 重点解决 Agent 间的信任问题和任务分配公平性，在去中心化金融 (DeFi) 和分布式计算场景中展示出实际应用潜力。
🔗 arxiv 去中心化 区块链 信任机制
OrgAgent: Organize Your Multi-Agent System like a Company
Apr 2026 2026-04-01 Yiru Wang, Xinyue Shen, Yaohui Han, Michael Backes, Pin-Yu Chen, Tsung-Yi Ho
提出公司式层级化多 Agent 框架，将协作分为治理层 (Governance)、执行层 (Execution) 和合规层 (Compliance)。 借鉴企业组织结构设计 Agent 角色和汇报关系，显著提升复杂任务的完成质量和一致性。
🔗 arxiv 层级化架构 治理 合规
Semantic Consensus: Process-Aware Conflict Detection and Resolution for Enterprise Multi-Agent LLM Systems
Apr 2026 2026-04 Vivek Acharya
面向企业级多 Agent 系统，提出基于语义共识的冲突检测与解决机制。通过过程感知的语义分析， 自动识别 Agent 间输出冲突并进行协调，确保企业工作流的正确性和一致性。
🔗 arxiv 语义共识 冲突检测 企业级

🧠 Agent 推理与规划 4 篇

Do Agents Think Deeper? A Mechanistic Investigation of Layer-Wise Dynamics in Sequential Planning
May 2026 2026-05-27
从机制可解释性角度研究 LLM Agent 在多轮规划任务中不同层的贡献。发现在 Agent 场景下， 模型深度利用模式与单轮推理完全不同：更深层在长期规划中承担更多"世界建模"和"状态追踪"职责， 这为设计更高效的 Agent 推理架构提供了关键洞察。
🔗 arxiv 机制可解释性 层间动态 世界建模
Efficient Agentic Reasoning Through Self-Regulated Simulative Planning
May 2026 2026-05-21 Mingkai Deng, Jinyu Hou, Lara Sá Neves, Varad Pimpalkhute, Taylor W. Killian, Zhengzhong Liu, Eric P. Xing
提出自我调节的模拟规划 (Self-Regulated Simulative Planning) 机制：Agent 自主决定何时及如何规划， 通过"世界模拟"生成候选方案并在模拟结果反馈下迭代优化。该机制在避免过度规划的同时维持高质量决策， 在多个 Agent 基准上显著超越 ReAct 等主流范式。
🔗 arxiv 模拟规划 自适应计算 世界模拟
MAP: A Map-then-Act Paradigm for Long-Horizon Interactive Agent Reasoning
May 2026 2026-05-13 Yuxin Liu, Ziang Ye, Yueqing Sun, Mingye Zhu, Jinwei Xiao, Zhuowen Han, Qi Gu, Xunliang Cai, Lei Zhang
提出"先建图再执行"(Map-then-Act) 的长周期 Agent 推理范式。Agent 先构建任务空间的显式认知地图， 包括子任务依赖关系和资源约束，再在地图引导下逐步执行。在复杂交互式环境中大幅减少无效探索和错误累积。
🔗 arxiv 长周期规划 认知地图 交互式推理
EEVEE: Efficient Test-Time Prompt Learning for LLM Agents
Jun 2026 2026-06
提出在测试时动态学习最优 Prompt 的方法 EEVEE，使 Agent 能在不重新训练的情况下 根据任务反馈自动调整指令和示例。在多个 Agent 基准测试中展现出即插即用的适应性提升。
🔗 arxiv Test-Time Learning Prompt 优化 自适应

🔧 Agent 工具使用 / Function Calling 4 篇

Tools as Continuous Flow for Evolving Agentic Reasoning
May 2026 2026-05-08 Tairan Huang, Siyu Shang, Qiang Chen, Xiu Su, Yi Chen
将工具调用从离散步骤转变为连续流 (Continuous Flow) 范式。Agent 在推理过程中动态地将工具调用 编织进思维链，形成推理-工具的连续梯度流。相比传统 step-wise 调用，在长链工具使用任务中 减少错误累积且泛化到未见工具的能力显著提升。
🔗 arxiv 连续流 工具编织 泛化能力
Concurrency without Model Changes: Enabling Async Function Calling in LLM Agents
May 2026 2026-05
提出无需修改模型即可实现异步并发函数调用的方法。通过在 Agent 框架层面引入并发调度器， 支持 Agent 同时发起多个独立工具调用并异步合并结果，大幅提升多工具任务的执行效率。
🔗 arxiv 异步调用 并发 效率提升
SkillSmith: Compiling Agent Skills into Boundary-Guided Runtime Interfaces
May 2026 2026-05-12 Duling Xu, Zheng Chen, Zaifeng Pan, Jiawei Guan, Dong Dong, Jialin Li, Bangzheng Pu
将 Agent 技能编译为边界引导的运行时接口 (Boundary-Guided Runtime Interfaces)。 通过静态分析技能的前置条件、副作用和资源需求，生成类型安全的调用接口， 避免了技能注入到系统提示中带来的 prompt 膨胀和安全隐患。
🔗 arxiv 技能编译 运行时接口 类型安全
An Executable Benchmarking Suite for Tool-Using Agents
May 2026 2026-05-10 Zhiqing Zhong, Zhijing Ye, Jiamin Wang, Xiaodong Yu
提出可执行的工具使用 Agent 评估套件，提供标准化 Web 和 CLI 工具环境， 支持对 Agent 的工具选择、参数填充和执行规划能力进行全面评测。 包含 200+ 工具覆盖多种真实应用场景。
🔗 arxiv 工具评估 Benchmark 可执行环境

🛡️ Agent 安全与对齐 4 篇

The Cold-Start Safety Gap in LLM Agents
Jun 2026 2026-06-05
发现并系统研究了 LLM Agent 的"冷启动安全缺口"现象：Agent 在对话会话初期最不安全， 经过几个正常的 Agent 任务后会显著变得更安全。提出 SODA (Safety Over Depth for Agents) 基准 来量化这一现象，并探讨了多种缓解策略。
🔗 arxiv 冷启动安全 SODA 基准 安全缺口
RUBAS: Rubric-Based Reinforcement Learning for Agent Safety
Jun 2026 2026-06-02 Xian Qi Loye, Qinglin Su, Zhexin Zhang, Shiyao Cui, Qi Zhu, Fei Mi, Hongning Wang, Minlie Huang
提出基于评分量规 (Rubric) 的强化学习框架 RUBAS，用于训练安全的工具使用 Agent。 通过多维度的安全评分体系对 Agent 轨迹进行细粒度评估，并使用 RL 优化策略。 在多种安全场景下显著降低不安全行为。
🔗 arxiv 安全 RL 评分量规 轨迹评估
On-Policy Self-Evolution via Failure Trajectories for Agentic Safety Alignment
May 2026 2026-05-12 Bo Yin, Qi Li, Xinchao Wang
利用 Agent 自身的失败轨迹进行在线自我进化来提升安全对齐。与仅关注最终回复不同， 该方法分析完整的工具调用轨迹（包括不安全的工具调用、注入指令等）， 通过 On-Policy Self-Evolution 逐步纠正不安全行为模式。
🔗 arxiv 自我进化 失败轨迹 安全对齐
Owner-Harm: A Missing Threat Model for AI Agent Safety
Apr 2026 2026-04-20 Dongcheng Zhang, Yiqing Jiang
指出现有 Agent 安全威胁模型中遗漏了"所有者伤害"(Owner-Harm) 威胁： 外部攻击者可能通过精心构造的输入（如恶意网页、文档）使 Agent 执行损害其所有者利益的操作。 提出分类框架和防护建议。
🔗 arxiv 威胁模型 所有者伤害 安全框架

📈 本周研究方向趋势总结

  • 🎯 Agent Harness 概念正在系统化： 多篇论文聚焦于 Agent Harness 的形式化定义、缺陷诊断和可复现性。 "Harness" 已从工程术语上升为学术研究对象，包含执行环境、工具接口、上下文管理、生命周期等子系统。 这对企业选型和框架设计具有指导意义。
  • 🛡️ Agent 安全研究全面加速： 安全方向出现了"冷启动安全缺口"、"轨迹级别安全评估"、"所有者伤害威胁模型"等多个新概念。 研究从对话安全转向端到端的 Agentic 行为安全，关注完整工具调用轨迹而非仅关注最终输出。
  • 🌐 去中心化与协议层成为新热点： 多个团队独立提出去中心化 MAS 方案和 Agent 通信协议。IoA (Internet of Agents) 的概念正在形成，包括能力公告、服务发现、信任层等基础设施需求。
  • 🧠 推理规划范式持续创新： 从 ReAct 式逐步推理到 Self-Regulated Simulative Planning（模拟规划）、 Map-then-Act（建图+执行）等新范式，关注点和创新点都在从"如何推理"转向"何时推理"和"推理什么"。
  • 💻 编程 Agent 走向深度仓库理解： SWE-Bench 之后的评估已超越"能不能修 Bug"，进入仓库探索能力 (SWE-Explore)、 规格推理 (SpecBench)、代码问答 (Code-QA-Bench) 等更细粒度的能力分析。
  • 🔧 工具使用进入"编译器"思维： SkillSmith 和 Tools as Continuous Flow 两篇论文分别从编译理论和连续统角度重新思考工具使用， 将工具从离散的 API 调用升级为更高级的抽象。
  • 🏗️ 企业级落地需要"可信基础设施"： RAILS (验证清算) 和 Confidential Computing (机密计算) 两篇论文明确指出， 企业级 Agent 部署需要超越模型能力的基础设施：验证、清算、隐私保护。
